	37. Walk me through Lambda's git flow. 

	1. First we fork the project we're working on so we don't modify the original file for other students.
	2. we can add a collaborator if we choose to, so they can give us feedback on our commits.
	3. then we clone a copy of the repo  to work with on our computer
	4. we checkout a branch and name it using our name as follows 'joshua-bartolomei'
	5. we complete our assignment on our computer and make sure to commit our work regularly.
	6. push the changes we made locally to the remote branch.
	7. make a pull request using our student branch aginst the fork we created in the beginning and then we submit it in canvas.
